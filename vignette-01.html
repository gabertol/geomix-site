Overview

This vignette validates the core geomix unmixing workflow using the DUMMY4 synthetic dataset, where true mixing proportions (A) and true source signatures (B) are fully known. Because ground truth is available, recovery accuracy can be measured directly with mean absolute error (MAE).

Dataset: 15 artificial mixtures (MIX1–MIX15) of three end-members (K = 3) across two complementary proxy blocks:

Block Type Description
DZ Distributional (KDE) Detrital zircon: 7 geochemical proxies per grain
BP Compositional (simplex) Bulk petrography: 5 mineral fractions per sample

The two blocks intentionally sample different grain-size fractions of the sedimentary load, providing complementary provenance signals.

Six methods are benchmarked:

  1. Baseline — NNLS in simplex space, joint blocks, inv_f weighting
  2. Entropy + inv_f — same, with sample-adaptive entropy weighting
  3. DZ only — single-block NNLS on distributional data
  4. BP only — single-block NNLS on compositional data
  5. CLR + NNLS — both blocks CLR-transformed, NNLS solver
  6. CLR + OLS — both blocks CLR-transformed, OLS solver

Methods 5–6 serve as foils: the CLR transformation breaks the linear mixing structure that holds in simplex space, and the benchmark quantifies the resulting loss in recovery accuracy.

2. Build data matrices

KDE normalisation uses y / sum(y) (sum = 1 per variable), which preserves the absolute scale difference between blocks that inv_f global weighting relies on. A global bandwidth (pooled across all samples) and a common evaluation grid are used per variable, matching the validation script exactly.

dz_vars <- setdiff(names(DUMMY4_DZ), "sample")
samples <- sort(unique(DUMMY4_DZ$sample))

# ---- local KDE helpers (matching validation script) -------------------------

get_evalpoints <- function(x, n = 129, probs = c(0.03, 0.97)) {
  x <- x[is.finite(x)]
  if (length(x) < 2) return(seq(0, 1, length.out = n))
  q <- quantile(x, probs = probs, na.rm = TRUE)
  seq(q[1], q[2], length.out = n)
}

default_bandwidth <- function(x) {
  x <- x[is.finite(x)]
  if (length(x) < 2) return(1)
  bw <- tryCatch(bw.SJ(x, method = "ste"), error = function(e) bw.nrd0(x))
  if (!is.finite(bw) || bw <= 0) bw <- 1
  bw
}

discretize_kde <- function(x, evalpoints, bw = NULL) {
  x <- x[is.finite(x)]
  if (length(x) < 2) return(rep(0, length(evalpoints)))
  if (is.null(bw)) bw <- default_bandwidth(x)
  dens <- density(x, bw = bw, from = min(evalpoints),
                  to = max(evalpoints), n = length(evalpoints))
  dens$y
}

5. Run unmixing — six methods

All methods use unmix() with normalize_blocks = "by_col" (the default), which normalises B and X by column L2-norms of B before solving for A, while computing the convergence loss on the original raw scale. This is essential for inv_f block weighting to function correctly across blocks with very different feature counts (DZ: 903 vs BP: 5).

clr_rows <- function(M, eps = 1e-6) {
  M[M <= 0] <- eps
  M <- M / rowSums(M)
  as.matrix(compositions::clr(compositions::acomp(M)))
}

DZ_clr <- clr_rows(DZ_mat)
BP_clr <- clr_rows(BP_mat)

run <- function(...) {
  fit <- unmix(...)
  rownames(fit$A) <- samples
  fit
}

6. Align end-members to ground truth

End-members are permuted using the Hungarian algorithm applied to Hellinger distances between estimated and true DZ signatures (B₁). For BP-only fits, alignment uses MAE on mineral proportions (B₂). For CLR fits, B is back-transformed to simplex before alignment.

hellinger <- function(p, q) {
  p[!is.finite(p)] <- 0
  q[!is.finite(q)] <- 0
  s_p <- sum(p); s_q <- sum(q)
  if (s_p <= 0 || s_q <= 0) return(1)   # máxima distância se KDE vazia
  p <- p / s_p
  q <- q / s_q
  sqrt(0.5 * sum((sqrt(p) - sqrt(q))^2))
}

align_via_B1 <- function(A_est, B1_est, B_true_dz, dz_vars, n_pts, A_true) {
  K_loc <- ncol(A_true)
  cost  <- matrix(0, K_loc, K_loc)
  for (h in seq_len(K_loc)) {
    for (k in seq_len(K_loc)) {
      d <- 0
      for (vi in seq_along(dz_vars)) {
        cols   <- ((vi - 1) * n_pts + 1):(vi * n_pts)
        d      <- d + hellinger(B_true_dz[h, cols], B1_est[k, cols])
      }
      cost[h, k] <- d / length(dz_vars)
    }
  }
  perm  <- as.integer(clue::solve_LSAP(cost))
  A_out <- A_est[, perm, drop = FALSE]
  colnames(A_out) <- colnames(A_true)
  list(A = A_out, perm = perm)
}

align_via_B2 <- function(A_est, B2_est, B_true_bp, A_true) {
  K_loc <- ncol(A_true)
  cost  <- matrix(0, K_loc, K_loc)
  for (h in seq_len(K_loc)) for (k in seq_len(K_loc))
    cost[h, k] <- mean(abs(B_true_bp[h, ] - B2_est[k, ]))
  perm  <- as.integer(clue::solve_LSAP(cost))
  A_out <- A_est[, perm, drop = FALSE]
  colnames(A_out) <- colnames(A_true)
  list(A = A_out, perm = perm)
}

clr_to_simplex <- function(M) {
  out <- t(apply(M, 1, function(r) as.numeric(compositions::clrInv(r))))
  out / rowSums(out)
}

al_base     <- align_via_B1(fit_base$A,     fit_base$B_list[["DZ"]],
                             B_true_dz, dz_vars, N_POINTS, A_true)
al_ent      <- align_via_B1(fit_ent$A,      fit_ent$B_list[["DZ"]],
                             B_true_dz, dz_vars, N_POINTS, A_true)
al_dz       <- align_via_B1(fit_dz$A,       fit_dz$B_list[["DZ"]],
                             B_true_dz, dz_vars, N_POINTS, A_true)
al_bp       <- align_via_B2(fit_bp$A,       fit_bp$B_list[["BP"]],
                             B_true_bp, A_true)
al_clr_nnls <- align_via_B1(fit_clr_nnls$A,
                             clr_to_simplex(fit_clr_nnls$B_list[["DZ"]]),
                             B_true_dz, dz_vars, N_POINTS, A_true)

mae <- function(e, t) mean(abs(e - t))
r2  <- function(e, t) cor(as.vector(t), as.vector(e))^2

12. Key results

The benchmark confirms the following properties of the geomix unmixing framework:

Joint vs. single-block: When blocks carry genuinely complementary information (DZ and BP sample different grain-size fractions here), joint unmixing with inv_f weighting outperforms both single-block approaches. The inv_f scheme correctly down-weights the DZ block (903 features) relative to BP (5 features), preventing the high-dimensional distributional data from dominating.

Simplex vs. CLR: CLR-transformed methods (CLR + NNLS, CLR + OLS) yield substantially higher MAE than their simplex-space counterparts. This is consistent with the theoretical incompatibility of CLR transformation with linear mixing: mixing is linear in simplex space, but the CLR map is non-linear with respect to mixing proportions, so NMF-style alternating solvers converge to systematically biased solutions.

Adaptive weighting: Entropy + inv_f does not outperform the Baseline on DUMMY4. This is expected — entropy weighting helps most when samples vary strongly in how well each block constrains the unmixing, which is not the case in this uniform synthetic dataset.
